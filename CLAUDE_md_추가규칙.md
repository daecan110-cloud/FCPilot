## Sprint 완료 테스트 규칙 (절대 생략 금지)

### 1. 변경 사항 체크리스트 자동 생성

매 Sprint(또는 Phase) 완료 시, 텔레그램 완료 보고에 **반드시** 아래 형식의 체크리스트를 포함한다:

```
📋 영민 테스트 체크리스트

이번에 추가/수정한 기능:
- [ ] {기능1} — {확인 방법}
- [ ] {기능2} — {확인 방법}
- [ ] {기능3} — {확인 방법}

이전 미테스트 항목 (누적):
- [ ] {이전 Sprint에서 못 한 항목}

테스트 방법: streamlit run app.py 또는 fcpilot-kr.streamlit.app
```

### 2. 체크리스트 작성 규칙

- 변경한 **모든** 기능을 빠짐없이 나열
- 각 항목에 **구체적인 확인 방법** 포함 (어떤 탭 → 어떤 버튼 → 뭐가 보여야 정상)
- 신규 기능뿐 아니라 **수정한 기존 기능**도 포함 (사이드이펙트 확인)
- DB 컬럼 변경 있으면 → "고객 등록 → 저장 → 에러 없이 성공" 항목 필수

### 3. 미테스트 항목 누적

영민이 테스트할 수 없는 환경(PC 없음, 시간 부족 등)이면:
- handoff.md에 `## 미테스트 항목` 섹션에 누적 기록
- 다음 Sprint 완료 시 체크리스트에 **이전 미테스트 항목**도 함께 포함
- 미테스트 항목이 5개 이상 쌓이면 → ⚠️ "테스트 먼저 해주세요" 경고 발송

### 4. Sprint 완료 선언 조건

Sprint 완료로 선언하려면 다음 **둘 다** 만족:
1. 자동 테스트 통과 (tests/ 실행)
2. 영민 수동 테스트 체크리스트 발송 완료

영민이 "확인" 답장하기 전까지는 **Sprint 완료가 아님**. 
다음 Sprint 시작 가능하지만, handoff.md에 "영민 테스트 대기 중" 표시.

## 작업 쪼개기 규칙

### 1. 한 번에 최대 3개 파일 변경

- 하나의 작업 단위에서 **3개 파일 이하**만 수정
- 3개 넘으면 작업을 쪼개서 중간 커밋
- 예: "고객 등록 수정 + 상담기록 수정 + 필터 추가" → 3개 각각 별도 커밋

### 2. P0 → P1 → P2 순서 엄수

버그픽스 설계서가 있으면:
- P0 전부 해결 → 커밋 → 테스트 체크리스트 발송
- 영민 확인 후 → P1 진행
- P0+P1 한꺼번에 하지 말 것

### 3. 큰 기능은 단계별 분리

예시 — "팔로업/동선 재구성" 같은 큰 작업:
```
Phase A: DB 조회 로직만 수정 → 커밋
Phase B: UI 표시 변경 → 커밋  
Phase C: 지도 연동 → 커밋
```
한꺼번에 하면 뭐가 깨졌는지 찾기 어려움.

### 4. 중간 보고

3개 이상 작업이 있으면 **중간 보고** 텔레그램 발송:
```
📌 중간 보고 (3/7 완료)
✅ BUG-01 고객 등록 수정
✅ BUG-02 상담기록 수정  
✅ BUG-04 전화번호 보정
⬜ BUG-05 간판 OCR...
```
